Refatoração: Fim do Shadow DOM no unkayOS

Por que remover o Shadow DOM?

  • Isolamento excessivo dificultava integração entre apps.
  • Debug e inspeção ficavam complexos.
  • Estilos globais não penetravam.
  • Eventos e comunicação entre apps eram problemáticos.

Nova Estratégia

  • Injeção direta: HTML/CSS/JS dos apps vão direto no DOM principal.
  • Namespace CSS: Cada app recebe um namespace automático, evitando conflitos de estilo.
  • Limpeza automática: CSS e elementos são removidos ao fechar o app.

Como ficou para desenvolvedores?

  • Apps: Não precisam mudar nada, o core faz o namespace.
  • Core: AppCustomUI e AppWindowSystem usam injeção direta + namespace.
  • Debug: Inspeção e eventos funcionam normalmente.

Vantagens

  • Código mais simples e fácil de manter.
  • Melhor compatibilidade e performance.
  • Facilidade para customizar e depurar.

Testes recomendados

  • Abrir apps, verificar UI e estilos.
  • Fechar apps, garantir limpeza de CSS.
  • Testar taskbar e menu iniciar.

Conclusão

O sistema ficou mais simples, flexível e robusto, mantendo isolamento via CSS sem as limitações do Shadow DOM.
